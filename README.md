Запуск проекта 
```
docker compose up --build
```

Создания админа через команду
```
docker compose exec app bash
php artisan create:admin
```

Авторизация в админке. Сделал через наименование в пути. При желании можно было настроить как префикс домента http://admin.site.ru
```
http://localhost/admin/auth/login
```

Реализовано 2 основные сущности для авторизации Admin и User. В текущем проекте сделано больше для демонстрации навыка -
можно было обойтись таблицей roles и задавать роли пользователей. Отдельная сущность, как по мне,
больше нужна в проектам с более сложной структурой. Например, где есть отдельные роли с пермишнами у админов.

Также упустил момент с Policy, его можно было бы добавить, если бы реализовывал просмотр страниц подписок у пользователя.
В текущей реализации нет путей в которые можно было бы перейти напрямую по ссылке.
